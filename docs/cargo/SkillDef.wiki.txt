One row per hero skill. 102 rows total in 2026-05-05: 30 production + 12 pseudo + 30 arena + 30 campaign. Skills are the top-level branches of the hero skill tree (e.g. <code>skill_assault</code>, <code>skill_archery</code>, <code>skill_logistics</code>); each has 1-3 mastery levels, and at each level the player gets to pick from a small pool of sub-skills ("perks") that further specialize the build.

A skill carries identity (id, variant, type), shared name + desc SIDs, and a count of mastery levels. Per-level data (overrides, offered sub-skills, bonuses) lives on [[Template:SkillLevelDef|SkillLevel]]. The sub-skills available from a skill live on [[Template:SubSkillDef|SubSkill]], recovered by scanning every level's <code>offered_sub_skills</code> list.

== Page layout ==
<code>Data:Skill/<id></code> is self-contained — it carries the skill's own row plus every sub-skill referenced by any of its levels:

# <code><nowiki>{{SkillDef | id=… | variant=… | skill_type=… | …}}</nowiki></code>
# <code><nowiki>{{TranslationDef | type=skill | target_id=<id> | …}}</nowiki></code> — name + desc shared across all levels
# N × <code><nowiki>{{SkillLevelDef | skill_id=<id> | level=N | …}}</nowiki></code>
# N × <code><nowiki>{{SkillLevelTranslationDef | skill_id=<id> | level=N | … }}</nowiki></code> — only when level overrides a name/desc
# M × <code><nowiki>{{BonusDef | parent_type=skill_level | parent_id=<id>_L<level> | … }}</nowiki></code>
# K × <code><nowiki>{{SubSkillDef | id=… | parent_skill_id=<id> | …}}</nowiki></code> — every sub-skill referenced by any level
# K × <code><nowiki>{{TranslationDef | type=sub_skill | target_id=<sub_skill_id> | …}}</nowiki></code>
# K' × <code><nowiki>{{BonusDef | parent_type=sub_skill | parent_id=<sub_skill_id> | …}}</nowiki></code>

The 77 sub-skills not referenced by any skill (8 test entries + ~69 arena legacy <code>*_old</code> entries) emit onto the catch-all page <code>Data:Skill/_orphan_sub_skills</code> instead.

== Field notes ==
* '''<code>id</code>''' is the source JSON id (e.g. <code>skill_assault</code>, <code>arena_skill_assault</code>, <code>campaign_skill_assault</code>, <code>skill_pseudo_1</code>).
* '''<code>variant</code>''' distinguishes production / arena / campaign / pseudo variants of the same skill family. Most analytical queries should <code>WHERE variant = 'production'</code>.
'' '''<code>skill_type</code>''' is one of <code>Common</code> / <code>Class</code> / <code>FactionDef</code>. Pseudo skills (the 12 <code>skill_pseudo_''</code> entries) carry no skill_type — they're internal mechanism skills not shown in the hero UI.
* '''<code>is_pseudo</code>''' mirrors the source <code>isPseudoSkill</code> flag. Pseudo skills typically have just 1 level and are used as reusable mechanic containers (e.g. movement bonuses applied uniformly).
* '''<code>name_sid</code> / <code>desc_sid</code>''' point at the L10n entries; <code>name</code> / <code>desc</code> are the resolved English mirrors for at-a-glance reading.
* '''<code>max_level</code>''' equals the length of the source <code>parametersPerLevel</code> array.
