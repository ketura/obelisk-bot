One row per adventure-map structure pulled from <code>DB/objects_logic/</code> — '''roughly 150 rows''' in the 2026-05-05 corpus across ~22 categories (resource piles, mines, dwellings, banks, chests, taverns, markets, portals, shrines, uniques, etc.).

The source data is heterogeneous (chests carry <code>variants</code> loot tables, hires carry <code>unitsData</code> + <code>bonuses</code>, mines carry
<code>resArr</code>, etc.) but most of that variance is deferred. This pass
captures the universal fields players care about — name, description, icon, basic combat / economy stats — plus a <code>source_path</code> pointer so editors can dig into the raw JSON for specifics the wiki doesn't yet surface.

Battlefield-side objects (<code>DB/field_objects/obstacles</code>, <code>sentries</code>,
<code>traps</code>) are out of scope for this table; they'll get their own
treatment.

== Page layout ==
<code>Data:MapObject/<id></code> carries:

# <code><nowiki>{{MapObjectDef | id=… | category=… | …}}</nowiki></code>
# N × <code><nowiki>{{TranslationDef | type=map_object | target_id=<id> | language=… | name=… | description=… }}</nowiki></code> — name+description, one row per language
# N × <code><nowiki>{{TranslationDef | type=map_object_narrative | target_id=<id> | language=… | description=… }}</nowiki></code> — narrative text (in the <code>description</code> column), one row per language, when a <code>narrative_desc_sid</code> exists

== Field notes ==
* '''<code>id</code>''' is the source JSON id (e.g. <code>mine_gold</code>, <code>barracks_human_3</code>, <code>celestial_sphere</code>, <code>tavern</code>, <code>portal_1</code>). No synthesis — the source ids are already globally unique.
* '''<code>category</code>''' is the source folder name under <code>DB/objects_logic/</code>. Used for filtering (e.g. "all dwellings" → <code>WHERE category = 'hires'</code>).
* '''<code>name_sid</code> / <code>desc_sid</code> / <code>narrative_desc_sid</code>''' follow the convention <code><id>_name</code> / <code><id>_description</code> / <code><id>_narrativeDescription</code> against <code>Lang/<locale>/texts/mapObjects.json</code>, surfaced for traceability. Display text for every language (English included) lives in the [[Template:TranslationDef|Translation]] table. ~94 of the ~150 rows have player-facing names; the rest (e.g. <code>tavern</code>, <code>outposts</code>, single-instance specials) have no <code>name_sid</code>, emit no <code>type='map_object'</code> Translation row, and rely on category for identification.
* '''<code>goods_value</code>''' is the in-source AI/economy weight (the "value" of taking this object). Universal — present on virtually every entry.
* '''<code>ai_value</code>''' / '''<code>custom_guard_value</code>''' appear on chests, hires, mines, resources — used by AI threat assessment.
* '''<code>guard_units</code>''' is the comma-joined list of <code><sid>:<amount></code> pairs (e.g. <code>esquire:2,crossbowman:2</code>). NULL when source has empty <code>guardUnits</code>.
* '''<code>fraction</code> / <code>tier</code>''' populated only for <code>category='hires'</code> (creature dwellings on the adventure map — ''not'' the city <code>Build_Tier_*</code> dwellings, which live on the Building table).
* '''<code>resource_name</code> / <code>resource_value</code>''' populated only for <code>category='res_mines'</code> (daily producers like <code>mine_gold</code>) and selected <code>res</code> rows. The richer <code>resArr</code> chance-table for resource-pile randomization (used in <code>category='res'</code>) is ''not'' extracted in this pass — the source path is on the row for editors who need it.

== Categories deliberately not extracted ==
* '''<code>cities/</code>''' — already modeled in the [[Template:BuildingDef|Building]] table.
* '''<code>items/</code>''' — these are placement metadata for artifacts on the map; the canonical Artifact data is in the [[Template:ArtifactDef|Artifact]] table.
* '''<code>blocks/</code>, <code>todo/</code>''' — terrain blockers and dev TODOs, not player-facing.
* '''<code>random_hires/</code>, <code>unit_upgrades/</code>, <code>town_gates/</code>, <code>win_condition_objects/</code>''' — generation/AI helpers and campaign-specific mechanics.

== Deliberately not extracted (within in-scope categories) ==
The rich category-specific payloads — chest <code>variants</code> loot tables, event-bank reward sets, hire <code>unitsData</code> weekly/bonus blocks, mine
<code>bonuses</code> arrays, magic-mine <code>bonuses</code>, market trade rates, etc. —
are out of scope for this pass. They'd each warrant their own side table (<code>MapObjectChestVariant</code>, <code>MapObjectEventBankReward</code>,
<code>MapObjectDwellingUnit</code>, etc.) following the Building pattern.

The <code>source_path</code> column on every row points editors at the source JSON file for those specifics until structured wiki coverage catches up.

== Related tables ==
* [[Template:TranslationDef|Translation]] — display text in every language (English included), joined on <code>id = Translation.target_id</code> with <code>Translation.type IN ('map_object', 'map_object_narrative')</code> and the desired <code>language</code> filter.
* [[Template:BuildingDef|Building]] — sibling table covering city-buildings (<code>DB/objects_logic/cities/</code>).
* [[Template:ArtifactDef|Artifact]] — canonical artifact data; map placements in <code>DB/objects_logic/items/</code> reference these by id.
* [[Template:UnitDef|Unit]] — <code>guard_units</code> entries reference Unit ids.
