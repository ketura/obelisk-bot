One row per (faction, tier) pair — 30 rows total (6 factions × 5 tiers). Captures the law-points threshold a player must accumulate before any tier-N law in a given faction becomes enactable.

In the 2026-05-05 corpus all six factions ship identical thresholds (<code>[0, 5, 15, 30, 50]</code>). Stored per-faction so a future patch can rebalance one faction independently.

== Page layout ==
<code>FactionLawTierDef</code> rows are emitted inline on the parent <code>FactionDef</code> page alongside the <code>LawTreePositionDef</code> rows.

== Field notes ==
* '''Primary key is <code>(faction, tier)</code>'''.
* '''<code>count_to_unlock</code>''' is sourced from <code>fractionLawsLines[tier-1].countToUnlock</code>.
* Tier 1 always has <code>count_to_unlock = 0</code> (no entry threshold); higher tiers progressively gate.
