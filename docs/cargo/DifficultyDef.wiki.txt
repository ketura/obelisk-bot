One row per game difficulty setting. 5 entries in 2026-05-05: Easy / Normal / Hard / Expert / Impossible. Each carries the balance inputs the engine actually uses: per-side starting-resource buckets and the global <code>neutral_power_multiplier</code> (the scalar applied to adventure-map encounter strength).

Two source-data quirks worth flagging up front:

* The source <code>nameSid</code> values (<code>EasyDifficultySid</code>, <code>NormalDifficultySid</code>, …) are ''not'' L10n entries — they don't resolve in any <code>Lang/<locale>/</code> file. We store them on the row for fidelity, but the canonical display name is the <code>id</code> itself.
* The source <code>descriptionSid</code> field carries literal English text ("This is an Easy difficulty setting."), not an L10n key. We store it as a plain <code>description</code> string column.

Because of those, this table doesn't carry a companion <code><nowiki>{{TranslationDef}}</nowiki></code> row — the source data ships in English only.

== Field notes ==
* '''<code>id</code>''' comes from the source <code>sid</code> field. Five fixed values; new difficulty tiers would be a balance change worth flagging.
* '''<code>neutral_power_multiplier</code>''' is the headline balance number. 1.0 is "Hard" (the canonical reference point); Easy halves encounter strength, Impossible doubles it.
* '''Resource columns''': source-side <code>alchemicalDust</code> is normalized to <code>dust</code> to match the canonical naming in [[Template:EntryDef|<code>Resource</code>]] (per <code>DB/res/resources_info.json</code>).
* '''Asymmetry''': on Easy, the player starts with 3× the resources the AI does; this flips on Hard+, with the Impossible AI getting 5× the player's gold. That's the ''primary'' difficulty mechanism — <code>neutral_power_multiplier</code> only modulates map encounters.
