'''One row per unit.''' All four attack slots — <code>default</code>, <code>counter</code>,
<code>alt</code>, <code>alt2</code> — collapse onto a single wide row, with each slot's
fields prefixed by the slot name. Pattern-passives like Sweeping Strike / Whirlwind / Dragonbreath aren't duplicated on every page; each slot references a shared [[Template:AttackPassiveDef|<code>AttackPassiveDef</code>]] row by <code>attack_passive_id</code>.

This pivot from one-row-per-slot to one-row-per-unit landed because (a) "list this unit's attacks" is the dominant query, and a single fat row beats <code>GROUP BY unit_id</code> joins; (b) per-slot rows would duplicate identity columns four ways for no benefit.

== Field notes — slot conventions ==
* '''<code><slot>_attack_type</code>''' is the only slot field that's always present when the slot itself is populated. If the unit has no slot of a given category, ''all'' <code><slot>_*</code> columns are NULL/absent.
* '''<code><slot>_passive</code>''' references [[Template:AttackPassiveDef|<code>AttackPassiveDef</code>]] by id. Most units leave this NULL (their attack pattern is plain <code>attack_single_x100</code> with no special passive). Patterns that resolve to a known passive get the id; unknown patterns fall back to a <code>pattern_passive_TODO_<pattern_sid></code> placeholder so the gap is visible.
* '''<code><slot>_pattern_sid</code>''' is usually omitted — the canonical pattern is implied by the passive_id. The bot only emits this when the unit's actual pattern differs from the passive's canonical sid (rare, e.g. <code>attack_reach_x2_x100_x100_x100_with_delay</code> is rank-2 of the Dragonbreath family; the bot stores the variant for forensic visibility).

== Field notes — defaults ==
The wiki template provides typed defaults; the bot omits any column whose value matches the default. Slot-specific defaults:

{| class="wikitable"
! Field !! <code>default_*</code> !! <code>counter_*</code> !! <code>alt_*</code> / <code>alt2_*</code>
|-
| <code>stat_dmg_mult</code> || <code>1.0</code> || <code>1.0</code> || <code>0.5</code>
|-
| <code>trigger_counter</code> || <code>true</code> || <code>false</code> || <code>false</code>
|-
| <code>damage_target</code> || <code>enemy</code> || <code>enemy</code> || <code>enemy</code>
|-
| <code>affect_target</code> || <code>enemy</code> || <code>enemy</code> || <code>enemy</code>
|-
| <code>damage_type</code> || <code>normal</code> || <code>normal</code> || <code>normal</code>
|}

Flat defaults (same regardless of slot):

* <code>is_armed_ability</code>, <code>dont_trigger_energy_regen</code>, <code>dont_use_energy</code>, <code>return_to_start_after_attack</code>, <code>never_disable</code> → <code>false</code>
* <code>cd</code> → <code>0</code> for default/counter, <code>-1</code> for <code>alt</code>/<code>alt2</code> is '''not''' the default (Crossbowman's plain melee fallback has cd=0). The template defaults <code><alt>_cd</code> to <code>0</code>; Fighting Style alts emit <code>cd=-1</code> explicitly.
'' <code>pattern_sid</code>, <code>passive</code>, <code>buff_''</code>, <code>temp_self_buff</code>, <code>multitarget_type</code>, <code>num_targets</code> → NULL

== Field notes — slot rarity ==
{| class="wikitable"
! Slot !! Population rate !! Notes
|-
| <code>default_*</code> || ~99% of units || Almost everyone has a default attack
|-
| <code>counter_*</code> || ~99% of units || Almost everyone has a counter
|-
| <code>alt_*</code> || ~50% of units || Ranged units' melee fallback, Fighting Style alts
|-
| <code>alt2_*</code> || ~3% of units || Medusa-tier (melee fallback + Arrow Barrage)
|}

Wide schema, sparse data: the typical row emits ~5 lines (the slot fields that deviate from defaults, plus identity columns).

== Sentinels ==
* <code><alt>_cd = -1</code> → once per battle.
* <code><slot>_buff_duration = -1</code> → infinite; <code>999</code> → until end of battle.
