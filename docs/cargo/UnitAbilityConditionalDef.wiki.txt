== Field notes ==
'' ''*<code>condition_*</code> fields decompose** the source JSON's positional triple <code>["unique_units_count", "ally", "demon"]</code> into three columns. This matches the structure of every conditional passive in the current corpus — the check function name, the side it applies to, and the parameter the check requires.
* '''<code>condition_check</code> enum values''' seen in the wild:
** <code>unique_units_count</code> — "you have N or more units of a faction"
** <code>unique_fractions_buff</code> — variant on the same theme
** More may surface in future patches; treat as open-string.
* '''<code>condition_target</code>:''' typically <code>ally</code> or <code>enemy</code>.
* '''<code>condition_value</code>:''' a faction id (<code>demon</code>, <code>human</code>, …) or similar discriminator. Always a string.
* '''<code>affected_stat</code> / <code>affected_stat_amount</code>''' describe the bonus granted when the condition fires. The current corpus has at most one stat per conditional passive, so a single pair suffices. If a future patch adds multi-stat conditionals, this would become a list — handle by extending the splinter or moving to a side table.
